# KOTO-0052: SDK Samples

- Status: done
- Type: docs
- Priority: P1
- Requirements: FR-SDK-1, FR-SDK-2, FR-SDK-4, FR-IME-1, FR-FS-2, FR-SIM-5, NFR-DEV-3
- Prerequisites: KOTO-0046, KOTO-0047, KOTO-0048

## Goal

Add a small set of source-authored sample apps that demonstrate the bytecode SDK
and serve as regression fixtures for the compiler, runtime, and simulator.

## Acceptance Criteria

- [x] Samples include at least hello text, input echo, counter/yield loop,
      sandboxed file note, IME playground, and dirty-rectangle drawing examples.
- [x] Each sample builds through the normal app build loop and launches in
      KotoSim.
- [x] The project harness verifies the generated bytecode for the samples.
- [x] Documentation explains what each sample demonstrates and which SDK calls it
      exercises.

## Notes

Samples are part of the developer experience, not throwaway demos. Keep them
small enough that failures point clearly to one subsystem.

Implemented in [KotoSDK Samples](../../spec/SDK_SAMPLES.md). The samples are registered
in `apps/apps.json`, build to committed `sdcard_mock/bytecode/sample_*.kbc`
fixtures, and launch through KotoSim by app ID.
