# KOTO-0039: Memo KPA Fixture

- Status: done
- Type: feature
- Priority: P1
- Requirements: FR-PKG-1, FR-PKG-2, FR-PKG-3, FR-SIM-3, FR-SIM-5

## Goal

Create a launchable `dev.koto.memo` package fixture for KotoSim that exercises
the memo app path through the same manifest, bytecode entry, icon, assets, and
sandbox permissions that a distributed `.kpa` app will use.

## Acceptance Criteria

- [x] `sdcard_mock/apps/memo.kpa.json` points at a real bytecode fixture and all
      referenced assets exist.
- [x] The fixture can launch from KotoShell through KOTO-0035 and call the memo
      editor/input/save paths needed for an MVP smoke test.
- [x] The fixture keeps assets sequential where required and remains compatible
      with the KPA packer prototype.
- [x] Missing or invalid memo fixture assets are caught by the project harness.
- [x] The launcher icon remains small and directly decodable by KotoShell.

## Notes

Depends on KOTO-0035, KOTO-0036, KOTO-0037, and KOTO-0038. The first bytecode may
be hand-authored or generated by a tiny fixture assembler; compiler ergonomics
are out of scope for this issue.
