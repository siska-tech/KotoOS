# KOTO-0046: Koto Language Compiler MVP

- Status: done
- Type: feature
- Priority: P0
- Requirements: FR-PKG-3, FR-RT-1, FR-RT-4, FR-SIM-5, NFR-DEV-3, NFR-DEV-4
- Prerequisites: KOTO-0044, KOTO-0045

## Goal

Implement the first host-side compiler from the selected high-level Koto app
language to verifier-valid `KBC1` bytecode. The compiler should be deterministic,
produce useful source-location errors, and support enough language features to
write small interactive apps without hand-authored assembly.

## Acceptance Criteria

- [x] The compiler parses source files with functions, locals, integer/boolean
      expressions, string literals, fixed buffers, `if`, `while`, `break`, and
      `return`.
- [x] The compiler emits deterministic `KBC1` bytecode directly or through the
      KOTO-0044 assembly/IR layer.
- [x] Generated bytecode passes `verify_kbc` under simulator runtime limits.
- [x] Compile errors include filename, line, column, and a readable diagnostic.
- [x] Tests cover successful compile/verify round-trips and rejection of syntax,
      type, undefined-symbol, and unsupported-feature errors.

## Notes

The compiler is a PC development tool, not a device runtime component. Keep the
first implementation narrow and auditable. A small language that is easy to
compile correctly is preferable to a rich language that hides memory and host-call
costs.

Implemented as the `tools/koto-compiler` crate (`compile`/`compile_to_asm` + a
CLI): `source -> lex -> parse -> codegen (kbc-asm text) -> KBC1`, with a final
`verify_kbc` guarantee. Because the KOTO-0033 verifier is a single linear pass
(not control-flow aware) and the VM shares one 16-slot local file, codegen
**inlines every function** (recursion rejected), emits **branchless comparisons
and logical operators**, routes `return` through a reserved slot, and shapes
host-call result folds to the verifier's success-shape model. Tests run compiled
programs through the VM (arithmetic, control flow, `%`, inlined calls, early
return, buffers, string draw, yield/resume) and cover syntax / undefined-symbol /
arity / non-buffer-index / recursion / missing-`main` errors. Built-in host-call
intrinsics are functional here; KOTO-0047 formalizes them as the documented SDK
prelude with named constants.
